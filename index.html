Lesson 1: Introduction to the Web and HTML

The Web

The web is a collection of HTML documents, images, videos, and other files. The bulk of these files is made up of the actual HTML documents, which can link to other files, including other HTML documents, thus explaining how the web earns its name.

A user participates in the web through the browser. The HTML documents and files of the web are hosted on servers. In order to retrieve these files, the browser makes an HTTP request to the servers via the internet.

HTML

HTML stands for HyperText Markup Language. It is made up of text content (what you see), markup (what it looks like), references to other documents (such as images and videos), and links to other pages. Plain text entered into an HTML document will usually be rendered as plain text by the browser. To make any changes to the way plain text is presented or arranged, markup in the form of HTML tags and elements must be used.

An element is basically a set of HTML tags surrounding some visible content. With a few exceptions, only the content is rendered by the browser. The tags themselves merely dictate how the content should be displayed or arranged. Some elements have no content, and thus have no closing tag as well. These are called void elements.

Computers Interpret Language Literally

Computers are often called "stupid" by programmers because unlike humans, computers cannot interpret ambiguous language. They can only interpret language literally. Therefore, a minor mistake, which a human could easily correct, could be disastrous for a computer.

In HTML, forgetting to include a closing tag at the end of an element that requires one could have unintended consequences. Most commonly, an HTML element without a closing tag could cause its effect to continue until the end of the document. So, instead of only a single word being emphasized, the entire document could end up being emphasized. Again, this is because the computer cannot infer upon the programmer's true intentions.

Inline VS Block

HTML elements are either inline or block. Inline elements are just text (or images) that often continue with the flow of the text on the current line. Block elements, on the other hand, are essentially invisible boxes or blocks that are stacked on top of each other.

Container elements, which serve no other purpose than to contain nested elements, are also either inline or block. The span element is inline, while the div element is block. These containers can wrap up code that can then be easily styled with CSS.

Document Structure

Given the severe consequences that minor mistakes may cause, HTML document structure should be organized in a way that promotes readability. For this reason, HTML code is indented, similar to how English is indented in a paragraph essay. Unlike English, however, an indentation level in HTML, as in programming languages, persists throughout an idea.

Every HTML document begins with a document type declaration. It is then followed by the html, head, and body elements. The html element contains the entire document. The head element contains the document's meta data. The body element contains the visible content and makes up the bulk of the document.

Lesson 2: Creating a Structured Document

Structure of Websites

Websites are broken down into three main components, each represented by a different language: HTML for structure, CSS for style, and Javascript for behavior. Each of these languages makes up the rules for how websites can be built and structured.

Much like a file directory, the browser builds a tree-like structure of an HTML document. Elements are nested within other elements, which ends up looking like an upside-down tree with the root at the top and nested elements branching down. This tree-like structure representation of the website by the browser is known as the DOM (Document Object Model).

Developer Tools

Using developer tools, such as those provided in Google Chrome and other modern browsers, developers can identify the structure of a website. An excellent learning tool, developer tools can reveal the DOM of a website. With this, developers can make temporary changes to a website's content and structure. Moreover, highlighting an element in the developer tools will reveal its corresponding position and presentation on the actual website.

Click here for an in-depth overview of Google Chrome's developer tools.

Boxes and Grids

Using developer tools, it is made clear that websites are based around boxes and grids. In fact, every element, including those that appear to be circles, are actually just boxes. The appearance of a non-box element is the result of using a style. The box-based nature of websites helps developers make changes to existing layouts. A grid lies underneath and makes sure that everything is aligned properly.

Design Mockup to HTML

With the boxed-based nature of websites in mind, a developer can take a design mockup and convert it to an HTML document. First, with a variety of options, a mockup is boxified; boxes are drawn around every visual element of the design. Next, using container elements, such as the div element, the boxified mockup is translated to HTML.

Lesson 3: Adding CSS Style and HTML Structure

CSS

CSS stands for Cascading Style Sheets. It is the language that applies styles to HTML documents. In this context, styles refers to presentation or appearance. Using CSS, a user can rearrange elements, change colors, change fonts and font sizes, etc. One of the biggest advantages of using CSS is that it helps keep structure and content separate from each other.

CSS earns its name from the fact that it is made up of style rules that cascade from general rules to the most specific rule. This is because CSS can receive style rules from different sources, such as the browser default style sheets, inline style sheets, external style sheets, and so on.

CSS Components

CSS style rules are made up of selectors, properties, and values. Selectors are the elements that are targeted by the style rules. These are commonly the elements themselves, classes, or IDs. In defining a style rule, a selector is then followed by a pair of curly braces which contains one or more sets of a properties and corresponding values.

A style rule can be defined for multiple selectors. Multiple elements with a desired similar style can be grouped into classes in HTML. Then, in CSS, these elements can be selected by class and styled together. One of the biggest advantages of this strategy is the prevention or reduction of repetition.

Classes and IDs

In HTML, elements can be assigned a class or ID. A class groups elements into a similar category, while an ID acts as a unique identifier. For this reason, only classes should be used for grouping together multiple elements. IDs should only be used for a single element. In CSS, the elements grouped into a class or ID can then be selected and targeted for a style rule.

External VS <style> VS Inline

There are three ways of adding CSS code to a web page. First, CSS can be added by using an external CSS file. Using this method, a <link> tag is added inside of the <head> element of the HTML document with a link to the CSS file. This is the recommended method, since it provides site-wide organization and ease of maintenance, especially with larger projects.

The second method for adding CSS code to a web page is by using the <style> element in the HTML document. The CSS code is then added within this element, just as it would be written in an external file. This method is appropriate for testing and small projects, but can lead to maintenance and repetition issues as a site expands or changes.

The third method for adding CSS code to a web page is by writing the style inline with the HTML. Using this method, a style attribute is added to the desired HTML element. Thus, style must be added for every individual element. This is the least recommended method for adding CSS code to a web page, for it introduces the potential of worst-case scenario maintenance and efficiency problems.

Break Down of the Box Model

Every HTML element is a box made up of certain properties: the content itself, padding, border, and margin. At the center of this box is the content which is surrounded or padded by the padding. The border then encapsulates both the padding and the content. Finally, on the outside, the margin provides space around the box. These properties may be modified using CSS.

Code, Test, and Refine

When coding, a useful strategy for reaching a final product is to code, test, and refine. Apart from boxifying and translating a design mockup to HTML, code is first written, much like a rough draft for a document, as an initial attempt at reaching a final product. Next, code is tested using a variety of methods. For example, in the context of developing a web page, the page can be tested by resizing the window, clicking on links, and using different browsers. Finally, code is refined by making changes that get the initial version closer to the final product.
